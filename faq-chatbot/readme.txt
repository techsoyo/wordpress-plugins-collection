A chatbot that automatically responds to frequently asked questions in eCommerce stores.

== Description ==

FAQ Chatbot for eCommerce is a simple yet powerful chatbot designed to assist customers with frequently asked questions in your online store. The plugin helps reduce the load on your customer support team by providing instant answers to common questions about shipping, payments, returns, and other policies.

= Features =
* Responsive chat interface with Tailwind CSS
* Admin panel to easily manage FAQs
* Keyword-based matching for questions
* Fallback responses for unknown questions
* Optional conversation history logging
* Easy integration via shortcode [faq_chatbot]
* Compatible with most WordPress themes and eCommerce platforms

= How It Works =
1. Install and activate the plugin
2. Add FAQs through the admin panel
3. Insert the chatbot on any page using the shortcode [faq_chatbot]
4. Customers can now ask questions and receive immediate answers

== Installation ==

1. Upload the `faq-chatbot` folder to your `/wp-content/plugins/` directory
2. Activate the plugin through the 'Plugins' menu in WordPress
3. Go to FAQ Chatbot menu in your admin panel to configure settings and add FAQs
4. Place the shortcode `[faq_chatbot]` on any page where you want the chatbot to appear

== Frequently Asked Questions ==

= Can I customize the appearance of the chat widget? =

Yes, you can modify the styling through the admin panel. The plugin uses Tailwind CSS classes that can be overridden with custom CSS if needed.

= Does this plugin work with all themes? =

The plugin should work with any properly coded WordPress theme. If you experience any compatibility issues, please contact support.

= Can I export/import my FAQ database? =

Currently, the plugin does not have built-in export/import functionality, but this feature is planned for future updates.

== Privacy Policy ==

This plugin does not collect any personal data by default. If the conversation history feature is enabled, the plugin will store user questions in the WordPress database. The plugin developer has no access to this data.
